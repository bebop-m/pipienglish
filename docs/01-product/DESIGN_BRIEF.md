## 一、这是什么产品

给 9 岁女孩皮皮定制的每日背单词 PWA，运行在 **iPad Pro 11 英寸**。以 **横屏 1194×834pt** 为唯一已确认视觉母版。其他 iPad Pro 横屏尺寸使用固定舞台等比缩放和背景延展；竖屏先显示横屏引导，除非以后另行设计并经小皮确认竖屏母版。

核心玩法：每天 10 分钟学习 4 个新词并复习。完成必修获得 2 颗鸡蛋，鸡蛋可用于单巢孵化、三档料理、场景贴纸和角色装扮；每 36 个完成日解锁一个可永久回访的新农场章节。世界观是一段以年为单位持续展开的温暖农场旅行，皮皮是农场主。

**情感基调（设计必须遵守）：** 永不惩罚、永远轻松。答错不出红叉、不闪红屏；“被抓走”的小鸡只是探头等待救援，不表现得可怜；所有反馈都是庆祝、陪伴或温和鼓励。

## 二、视觉风格基准（最高优先级）

F4 已完成视觉校准。以下文字负责解释 F4，不能被用来重新诠释或替换 F4。完整规则见 `F4_VISUAL_SYSTEM.md`，母版文件见 `F4_BASELINE_MANIFEST.md`。

参考图由 9 岁的皮皮本人选定，是不可偏离的视觉基准。其中当前仓库已正式锁定两张最重要的原始参考图：

- `docs/reference-images/chick-character-style-reference.png`：小鸡角色造型与画风参考板；
- `docs/reference-images/pwa-icon-farm-master.png`：农场小鸡参考，同时是小皮确认的 PWA 桌面图标源母图。

完整用途、哈希和使用限制见 `docs/reference-images/README.md`。其余人物参考如后续确认原始文件，也应按同一方法归档后才能成为长期基准。

原始参考图的职责：

- **前两张参考图：** 提取小鸡角色造型、粗手绘轮廓、上色方式、农场场景和整体氛围。
- **第三张参考图：** 只提取小女孩的外形与气质，不继承其中的背景、材质或其他角色画风。
- 参考图是**绘画语言基准**，不是只供参考题材。不能仅保留“小鸡、草地、谷仓”等内容元素，却改成另一种儿童插画风格。

### 2.1 风格的一句话定义

**日系 Kawaii 手绘贴纸农场：全幅柔软水粉农场背景 + 粗深棕手绘描边的独立透明角色与互动道具 + 奶油纸牌式、圆润且可落地的学习 UI。**

“贴纸感”来自角色与柔软背景之间的层次差和可独立移动的资产组织方式，不来自统一的白色贴纸外框，也不等于扁平矢量卡通或贴纸收集册布局。F4 不是泛称的“绘本感”；不得只根据这类宽泛风格词另做一套画面。

### 2.2 整体绘画语言

- 使用**不透明水彩 / 水粉**的视觉质感，允许局部结合蜡笔、粉彩和干笔刷效果。
- 画面保留细腻纸张颗粒、颜料叠色、轻微色差、干笔刷边缘和不完全规整的手工痕迹。
- 色彩采用**高明度、适中偏低饱和度**的柔和配色，不要灰暗，也不要高饱和荧光色。
- 造型圆润、简化、稚拙，避免精准几何图形、锐角、机械对称和过度精修。
- 画面应像一个可自由活动的手绘贴纸农场，同时保留明确、真实可实现的学习 App 控件；不能退化成整张插画截图，也不能只在常规网页 UI 后面铺一张卡通背景。
- 所有角色必须属于同一套绘画系统：线条粗细、五官复杂度、上色方式和材质一致，不能像来自不同素材包。

### 2.3 角色绘制规则

- 粗描边只属于母鸡、小鸡、小皮，以及极少数需要强调的贴纸式元素；已废弃角色不再作为视觉规则参考。
- 描边使用深棕黑，不要纯机械黑。F4 UI 基准墨色为 `#302925`；角色轮廓的颜色、粗细和抖动以 `chick-f3.png`、`mother-f3.png` 和 `xiaopi-f3.png` 的实际像素为准，不得仅用一个统一 CSS 描边数值重画。
- 轮廓必须有自然的粗细变化、轻微抖动、圆润转折和落笔感；禁止均匀、光滑、精准闭合的矢量描边。
- 角色以大头、圆身体、短小肢体和极简五官为主。眼睛、嘴巴、腮红都要控制细节，保持幼拙感。
- 角色内部以柔和的不透明色块为主，允许很淡的水粉晕染和局部深浅不均；不要复杂体积光影。
- 小皮虽然来自第三张人物参考图，但必须重新翻译进小鸡的绘本画风：简化头发、眼睛、服装和配件，不能成为细节更多、画法更精致的另一类 Q 版角色。
- 除非参考图明确出现，否则不要给角色统一添加白色贴纸边框。

### 2.4 背景与场景绘制规则

- 天空、云朵、草地、灌木、花朵、栅栏、谷仓和小路**不使用深色黑描边**。
- “无黑色描边”不等于“平滑矢量色块”。背景必须有水粉叠色、干笔刷边缘、纸张颗粒和局部手绘不均匀感。
- 天空使用柔和的晴空蓝水粉色面；可以有细微色差，但禁止大面积数字渐变和发光效果。
- 云朵像用白色水粉或海绵笔拍出来，边缘蓬松、略有缺口，不要画成规则图标。
- 草地由数层柔和绿组成，加入少量短笔触、叶片和白色小花；避免一整块鲜绿色渐变。
- 谷仓、栅栏和道路应明显保留手工涂画感，线条可以稍歪、色块可以不完全填满，但不能脏乱。
- 背景细节服务于温暖、可探索的农场氛围，不能抢过角色，也不能简化成几条色带。

### 2.5 UI 与绘本场景的融合

- 首页首先是一幅完整、可进入的农场绘本，UI 是轻量交互层，不是占据画面中心的大型网页弹窗。
- 输入、按钮、进度和信息尽量转译成场景内物件，例如奶油色手写名牌、小木牌、布标、篮子标签或鸡舍告示牌。
- 卡片和按钮默认不使用粗黑边。可使用奶油色纸张边缘、2–3px 柔和棕色细线、轻微纸片阴影或色块层叠。
- 禁止满屏黑框、硬实色投影、悬浮玻璃卡片和标准 SaaS 式面板。
- 主要按钮可以醒目，但仍需柔软、圆润，与绘本材质一致；不要使用塑料高光或强渐变。
- 装饰符号（爱心、闪光线、汗滴、欢呼线）使用无描边的白色或浅色简笔画。
- 字体使用系统圆体优先的字体栈，标题可粗，但避免所有文字都使用 900 字重。正文必须保证孩子在 iPad 上易读。
- 家长入口保持清晰但低调，不与孩子的主要任务入口争夺注意力。

### 2.6 横屏构图基准

- 标准画布为 **1194×834pt**，不是 16:9。关键角色、文字和点击区域需避开设备安全区，外边距建议不小于 32pt。
- 首页采用全幅农场构图，角色和交互形成一个明确的故事场景。避免人物分别孤立在左右两端，也避免角色被大卡片遮住。
- 首次状态中，母鸡是主要叙事角色，小皮在旁边与母鸡产生动作或视线互动。
- 角色组应成为第一视觉焦点，农场和谷仓建立环境层次；避免天空占据近半画面却没有叙事作用。
- 起名交互应融入母鸡附近的场景物件，不使用居中的大型注册弹窗。
- F4 横屏核心舞台不得为不同 iPad 单独重排。竖屏不是简单缩放或裁切横屏；在独立竖屏母版确认前，显示温和的横屏使用引导。

### 2.7 明确禁止的视觉方向

- 扁平矢量插画、企业儿童卡通、通用教育 App 素材风。
- 仅给平滑矢量路径添加统一噪点，冒充水粉手绘质感。
- 3D 渲染、塑料光泽、玻璃拟态、霓虹发光和写实材质。
- 动漫赛璐璐阴影、复杂高光、写实毛发和精细鼻口结构。
- 平滑数字渐变、大面积鲜艳纯色、精准几何云朵和规则波浪草地。
- 角色之间描边粗细、眼睛画法、细节密度或材质明显不一致。
- 居中大白弹窗遮挡农场、满屏粗黑 UI 边框、硬质投影。
- 过度精致的商业贴纸效果、统一白色外框和无手工痕迹的素材库风格。

## 四、屏幕清单（核心学习 6 块 + 长期农场附属界面）

### 4-1. 首页·农场（F4 已确认，正式接入优先）

首页是一幅全屏、可互动的水粉农场绘本。每个 36 日章节有独立背景与主题，但必须保留任务板、单巢鸡窝、救援入口、角色活动区和贴纸安全区。母鸡、小皮与小鸡群处于同一故事场景中。

当前场景同屏最多 40 只；最多 8 只由小皮选择的“最喜欢”优先显示，更多小鸡收进本场景“鸡舍 ×N”。三个状态共享当前旅程场景，但氛围和主要交互发生变化：

- **首次：** 农场里还没有小鸡，但不是空白页面。母鸡与小皮同屏，通过场景中的名牌或小木牌为母鸡起名。不要使用居中大弹窗。
- **未完成：** 农场已有小鸡；母鸡或小皮邀请孩子开始当天学习。主要入口为醒目的场景物件，文案是「开始今天的单词！」。
- **完成：** 农场进入轻度庆祝状态；主要入口变成连胜天数展示牌，同时解锁手写游戏入口。庆祝要温暖、有生命力，但不要使用强烈闪烁和满屏粒子。

常驻元素：

- 小鸡群聊：点一只小鸡，它和周围 2–3 只同时冒出漫画气泡，显示英文与中文。
- 单巢孵化：只显示一个蛋位，完整蛋、裂纹和破壳状态共享稳定锚点；孵化中常驻显示友好的剩余时间，不做压力式进度条。
- 待救小鸡入口：使用 F4 已确认的独立救援篮图片，并显示 `×N` 数量；表现为温和的小提示，不营造紧张感。
- 旅行路牌与地图入口：不遮挡主学习入口；新章解锁后让小皮选择现在出发或以后再走，不能制造倒计时或催促感。
- 贴纸与装扮：已摆放贴纸要像场景原生物件，但仍是独立透明资产；小皮和母鸡装扮遵守槽位、遮挡层和脚底锚点。
- 家长页小入口：位置稳定、可发现，但视觉权重低。

### 4-2. 学习流卡片

复习和新词共用一条卡片流，顶部是母鸡走向鸡窝的进度条：

- 听看卡：F4 本地单词插图 + 单词 + 释义 + 例句 + 文字清晰的重听按钮
- 描红卡：四线三格 + 淡灰字模 + 手写画布 + 清除 / 重听 / 写好了
- 选择题卡：单词 + 4 个中文选项；确需图像辅助时使用同一套 F4 本地单词插图
- 听音卡：大喇叭按钮 + 4 个选项
- 默写卡：中文 + 标准英文输入框 +「偷看一眼」按钮；键盘或 iPad 系统手写识别均可，按最终英文文本客观判定
- 所有答题卡都有一个不显眼但顺手的「忘记了」按钮

### 4-3. 结束页

母鸡妈妈竖大拇指说“真棒！”+ 战报（今天认识 X 个新朋友、复习 Y 个老朋友、连胜 Z 天）+ 2 颗鸡蛋奖励 +「回农场」按钮。结束页继续使用批准的标准母鸡造型，不要求同步首页衣柜装扮。

### 4-4. 手写小游戏

中文 → 输入英文，10 题一轮，显示轮内进度。输入可来自键盘或系统手写识别，应用只判断最终英文文本。每日前 10 轮结算为 1 颗生鸡蛋入库，第 11 轮起仅显示继续练习，不强迫当场消费。

### 4-5. 救援（补错）界面

简单列表（待救小鸡 ×N）→ 逐词练习：中文 + 本地发音按钮 +「偷看一眼」+ 英文文本输入 → 写对后小鸡回家动画。

### 4-6. 家长页

学习统计 + 数据备份按钮。保持朴素、清晰，给家长使用，不需要儿童页同等程度的装饰。

### 4-7. 长期农场附属界面

- **章节地图：**已解锁场景清楚可达，未打包章节不显示虚假入口；旧场景回访与当前旅程要有明确视觉区别。
- **鸡舍：**可浏览当前场景全部居民，点按仍可说单词，并能一步操作小星星；第 9 只最喜欢使用简洁替换选择。
- **厨房：**在同一空间呈现 1 蛋单份煎蛋、5 蛋野餐餐盘、20 蛋庆典大餐。三者都强调动画和陪伴，不显示属性收益。存在未享用料理时出发，需要清晰提供“先去享用 / 收好鸡蛋再出发”。
- **贴纸目录与摆放：**完整章节为 4 小、3 中、2 地标；需显示价格、拥有状态和摆放层级，不做限时、折扣、缺失剪影或真实货币入口。
- **衣柜：**小皮三槽、母鸡两槽；换装预览必须能看清层级和遮挡，装备、卸下、跨场景切换均不出现收费暗示。
- **章节庆祝与合照：**36 日解锁庆祝免费；20 蛋庆典大餐是独立可重复活动，章节首次可生成一张永久合照，不能混成旅行门票。

## 五、硬性技术约束

- **离线 PWA：** 不得引用 CDN 字体、外链图片或其他在线资源。所有字体、插画和图标必须随项目本地打包。
- **F4 混合素材方案：**背景使用本地 RGB PNG/WebP；角色、可动物件和有手绘身份的场景物件使用透明 PNG/WebP；UI 结构、动态文字和常规控制使用 HTML/CSS。不得把角色改造成通用矢量图或字符图标以追求工程统一。
- **章节核心包：**每章必须离线打包背景、地图缩略图、免费路牌、3 件贴纸、2 款异色小鸡、1 款特殊小鸡和 1 件主题角色装扮；发布时始终预置未来两章核心包。扩展资产可后补但不得阻塞旅行。
- **角色分层：**小皮和母鸡常态母版统一使用 `back/body/front` 合成顺序，并为每件装扮记录目标角色、槽位、脚底/中心锚点与适用姿势。不得用整张新角色图覆盖后伪装成可组合装扮。
- 1194×834 核心画布的生产背景建议至少提供 2× 资产（2388×1668px）。高清重制必须保持相同构图并经 A/B 验收，在确认前继续使用已锁定原图。
- 小型功能控件优先使用清楚的文字、CSS 或已批准的本地 PNG/WebP；不得引入另一套图标画风。
- 位图资产需控制体积，优先使用 WebP；需要透明背景且 WebP 效果不理想时可使用 PNG。
- 动效优先使用 CSS `transform`、`opacity` 和 `keyframes` 实现，并在 `docs/02-visual/F4_VISUAL_SYSTEM.md` 与对应实现中标注名称、时长、缓动和触发条件。
- 手写画布区域只需预留容器样式，画布本身由工程实现。
- 所有主要点击目标在 iPad 上不小于 44×44pt；动画不得影响输入和阅读。

## 六、交付格式

1. `design-tokens.css` —— CSS 变量全集：颜色、字号阶梯、圆角、描边、阴影、间距和动效参数。
2. `screens/*.html` 或对应 React 页面 —— 每屏能在 1194×834 固定舞台独立验证，并使用真实本地素材；不得用整张截图冒充可交互页面。
3. `assets/backgrounds/*` —— 横屏农场背景和复杂水粉场景，使用本地 PNG/WebP。
4. `assets/characters/*` —— 角色及状态变体，使用同画布、同脚底锚点的透明 PNG/WebP。
5. `assets/ui/*` —— 有手绘身份的场景物件和需要图像表达的功能图标使用透明 PNG/WebP；普通控制由 HTML/CSS 实现。
6. 视觉回归资料 —— 1194×834 黄金截图、iPad 11/13 英寸截图、差异报告和资产哈希。
7. 设计说明 —— 更新 F4 专项文档、动效参数和新增资产需求单，不另建互相冲突的风格说明。

类名需语义化，例如 `.farm-chick`、`.farm-name-sign`、`.quiz-option`。不要把零网络依赖与所有资源内联混为一谈。
