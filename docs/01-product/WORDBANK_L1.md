## 2. 字段与文件结构(SPEC §4.4)

```text
src/domain/words/
  types.ts        Word 接口 + PACKS 注册表(数组顺序 = 投放顺序)+ definePack 构造器
  pack-*.ts       每包一个文件;新增词包 = 新文件 + index.ts 尾部追加
  index.ts        WORDS(投放顺序拼接)/ WORD_MAP
  legacyIds.ts    已发布 ID 快照(现为 v0.1 的 88 个);只增不改
  quality.ts      质检规则(纯函数)
  words.test.ts   质量门禁(CI 强制)
```

字段:`id / word / ipa(美式,含斜杠)/ meaning(儿童语言)/ emoji(仅开发壳与家长页)/
sentence(L1 ≤8 词)/ sentenceCn / pack / level / imageAssetId?(F4 插图,Codex P1 回填)`。

一词多义(如 orange 橙子/橙色、watch 看/手表)只保留一个最贴近生活的义项与词包,
第二义项到 L2 阶段再以新 ID 收录(如 `orange-color`),不改旧词条。

## 3. 质检门禁(`checkWordBank`,测试红即不能合并)

**errors(阻断)**:重复 ID / 空字段 / 音标非 `/…/` 格式 / 词包未注册 / 层级非法 /
例句不含该词(容忍复数、三单、过去式、进行时、比较级、y→ies 等词形变化)/ 例句超长(L1>8 词)。

**warnings(排期,不阻断)**:释义疑似混入英文 / 同包重复释义(干扰项抽取已按释义去重)/
同包同级词数 <4(选择题干扰项池,将回退全库同级)/ 缺 F4 插图统计。

**旧记录映射**:`legacyIds.ts` 快照 ⊆ 词库(测试强制);`findOrphanIds()` 亦可在家长页
用 cards 表的真实 wordId 运行,检测"库里查无此词"的孤儿记录。

## 4. 爸爸抽查流程与后续批次

- 抽查:直接读 `pack-*.ts`(一行一词,含例句与翻译);改错就地改字段,**不改 id**;
  跑 `npm test` 即完成回归。发布(推送 main)后如需删词,只能"停止投放"(移出 index 顺序需慎重),
  不删 legacyIds 中的 ID。
- 已规划批次(按需追加,每批一个新 pack 文件):
  1. **「刀与妖怪」**(鬼灭主题:sword/demon/breathe/slash…,SPEC §4.2,v1.0 里程碑);
  2. **「小小词」**(Fry 高频功能词:I/you/this/in/on/and…,配合 5–6 年级"中→英"产出题型再投放);
  3. **「太空」「魔法城堡」**(SPEC §4.2 已规划主题);
  4. L1 剩余生活词补充至 ~1500(随皮皮消耗进度滚动生成,始终保持 ≥3 个月余量)。
