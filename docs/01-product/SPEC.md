# 皮皮のEnglish — 产品规格 v1.0

> 一个为皮皮(2026 年 9 月上四年级)定制的每日背单词 iOS PWA。
> 终极目标:约 7 年后托福高分(词汇量 9,000+);产品目标:**让背单词 7 年里从未变成负担**。

## 0. 最高设计原则(冲突时的裁决依据)

1. **最低可持续速度**:每天 4 个新词 + ≤6 个复习(2026-07-17 爸爸调整:25→6,复习 6 + 新词 4 与写词游戏每轮 10 题一致),总时长 ≤10 分钟。这是硬上限,不是下限——状态好也不许多学。
2. **永不惩罚**:没有死亡、没有清零、没有欠账展示。缺勤后回来,今天的任务永远是 10 分钟能完成的量。
3. **一切激励指向拥有和成长,不指向比较和损失**:收集、进化、累计;不做排行榜、不做每日签到奖励、不做金币商城。
4. 所有设计决策冲突时,一律向"更轻松、更宽容"倾斜,而不是"更高效"。

## 1. 用户与角色

- **皮皮(学习者,总设计师)**:每日 10 分钟学习;自称"小屁股"。
- **爸爸(开发者/管理员)**:维护词库、看周报。
- **妈妈(学生)**:每周被皮皮考试的对象,知情且配合"偶尔答错"。

## 2. 屏幕框架与核心循环(2026-07 显示框架定稿)

### 2.0 导航总图

```
农场(首页)──→ 今日学习(必修)──→ 结束页(屁股超人+战报)──→ 回农场
   │ 完成当日必修后解锁
   ├──→ 手写小游戏(选修,10 题)──→ 得煎蛋喂小鸡 ──→ 回农场
   └──→ 家长页
```

### 2.1 首页 = 小鸡农场

- 农场即首页:母鸡 + 历史小鸡 + 场景,热闹可爱。
- **未完成态**:大大的「开始今天的单词!」入口;**完成态**:入口变成连胜展示板
  (连续天数),并解锁手写小游戏入口;**首次使用**:空农场(只有母鸡)+ 起名。
- **小鸡群聊**(2026-07 改版,替代"每只小鸡住固定词"):点任意小鸡 →
  被点的那只头顶漫画气泡说一个词(英文 + 中文 + 播发音),周围 2~3 只
  同时冒泡显示别的词(不抢音频)——农场七嘴八舌。词按**陈旧度加权**抽取
  (§2.4 同一套权重),并更新"上次见面时间":逛农场 = 无感复习。
- **待救小鸡入口**(错题呈现,见 §2.5):首页角落使用 F4 独立救援篮图片并显示 `×N`,
  只显示有几只小鸡"被抓走";点击进入救援(补错)界面。不做复杂场景。
- 同屏最多画 ~40 只小鸡,其余收进"鸡舍 ×N";里程碑解锁场景扩建(§3.1)。

### 2.2 今日学习(必修,FSRS 驱动)——听/看先行,写/读跟上

1. **复习队列**(FSRS 到期,≤6;安排与呈现见 §5.3)。
2. **新词 ×4**,每词三步:
   - ① 听看卡:图先出,发音自动播 2 遍,**词形延迟半秒出现**(音先于形);
     例句;末尾提示"跟着小鸡读一遍"(不判分);
   - ② 描红:四线三格(同语文练字册格式)超大淡灰字模,粗蜡笔质感笔迹,
     写完字模淡出只留她的字;常驻:重听 / 清除 / 写好了;
   - ③ 选择题自测(4 个中文选项,客观判分)。
3. **收尾·今日默写**(2026-07 爸爸提议):4 个新词全部学完后,各凭记忆
   手写一遍——显示中文 + 播发音,可"偷看一眼",写完对照答案自我修正,
   不评分。间隔 5~8 分钟后的提取,比紧跟描红的仿写增益高得多,故原
   "仿写"步骤取消、书写时间挪到此处。它同时是每天的**签到仪式**
  ("今天的 4 个新朋友,名字各写一遍"),字迹存入当天孵化的小鸡
   (v0.3 点小鸡可看"那天皮皮写的字",数据 v0.2 起就存)。
4. 每完成一项,母鸡进度条前进;答错的题排到队尾当场重考,直到全部通过。
5. 书写原则:**次数分散**——当天描红 + 收尾默写各一遍,之后随复习熟练度
   升级为默写复习,不搞一天抄十遍。
6. 每日总时长目标 10~13 分钟(含手写)。

### 2.3 结束页

- 屁股超人三阶段动画(登场/说话/放屁飞走),台词照旧。
- **战报**:今天认识 X 个新朋友、复习 Y 个老朋友、连胜 Z 天。
- 「回农场看看」按钮。庆祝只在这里发生一次,农场完成态只留常驻连胜板。
- 完成 → 获得 1~2 颗蛋进入鸡蛋库存(分配与 24 小时孵化规则见 §3.1)。

### 2.4 手写小游戏(选修,完成必修后解锁,不限轮数)

- 每轮从已学词库抽 **10 题**:显示中文 → 手写英文 → 对照自评。
- **陈旧度加权抽词**:每个词记录"上次见面时间"(必修复习/写词游戏/点小鸡,
  取最近一次,全系统共享);抽中概率 ∝ (距上次见面天数 + 1)²。
  久未谋面的词显著更容易被抽中;昨天刚见过的几乎抽不到。
- 游戏结果**不写入 FSRS 调度**(自评 + 无限次游玩会污染排程),但会更新
  "上次见面时间"——游戏是 FSRS 之外的第二张安全网,专捞陈年旧词;
  玩得越多,必修复习越轻松,正循环。
- 奖励:完成一轮得 1 个煎蛋,喂给任意小鸡(开心转圈动画)——皮皮方案一的
  煎蛋互动在此回归,而且是她自己赚来的。
- **不限轮数**,想玩多少轮玩多少轮(2026-07 爸爸定):反正是学单词。
  加练欲望全部导向肌肉记忆通道,新词 4 个的硬上限不变。
- **没有"再来一组"新词的入口,任何界面都没有。**

### 2.5 忘记与救援(小鸡救援机制,2026-07 爸爸提议)

- 所有答题卡(选择题/听音/默写,含写词游戏)都有**"忘记了"按钮**:
  点击 → 温和展示答案 → 跳过(不队尾重考)→ 该词进入救援队列。
  它给 FSRS 提供最干净的 Again 信号(免得孩子乱猜污染数据)。
- 每个救援队列里的词 = 一只"被抓走"的小鸡;**首页仅以角标显示数量**
  (F4 救援篮图片 + `×N`),不做反派角色、不做复杂场景(2026-07 爸爸定)。
- **补错(救援)界面**:简单列表逐词——显示中文 + 播发音,可"偷看一眼"
  → 手写正确 → 小鸡被放回,回家动画。救援不评分、不入 FSRS(该词按
  Again 排程明天自然再来),只更新"见面时间"。
- **软化三条款(不可违反,守护"永不惩罚"底线)**:
  ① "被抓"呈现必须轻:无具象反派、无受苦画面,小鸡只是探头"等你来救";
  ② 永远可救、无时限、必定成功(偷看无妨,写出来本身就是目的);
  ③ 农场小鸡总数永不减少,被抓的只是换了个展示位。
- 与重考规则的分工:**选错 → 队尾当场重考;点"忘记了" → 跳过进救援**。

### 2.6 全局音频原则(磨耳朵)

- **任何一次书写完成**(描红/收尾默写/默写复习/写词游戏/救援补写)
  → 自动播放该词发音一遍;
- 选择题/听音题答对 → 也播一遍;
- 刚写完/刚答对的瞬间是音形绑定的黄金窗口,此规则全局生效、无例外。

### 2.7 记忆模型(为什么单日 4 步不需要"记住")

单日 4 步的任务是**高质量编码 + 一次成功提取**,不是记住。记住来自其后
一个月 6~8 次 FSRS 安排的重逢(1/2/4/8/15/35 天式的扩张间隔),外加点小鸡、
写词游戏的随机重逢。**中间忘掉是机制的一部分**(合意困难:快忘时费力想起,
增强最大);答错=这次复习来得正是时候,这句话要讲给皮皮和妈妈听。

## 3. 激励系统

### 3.1 母鸡与农场(核心养成)

- 常驻一只母鸡(皮皮命名),学习时在旁边啄米/打鸣。
- **蛋经济(F4 定稿 + 2026-07 爸爸裁决)**:完成当日必修 → 获得 1~2 颗蛋进入
  **鸡蛋库存**;每颗蛋由皮皮亲自分配(F4「鸡蛋要怎么用?」面板):
  **孵化**(放入孵化棚,共 3 巢位)或 **煎蛋**(喂小鸡)。
- **孵化固定 24 小时**:放入后满 24 小时破壳成小鸡;**不显示倒计时条**,
  点击孵化中的蛋才显示剩余时间。蛋永不过期、永不没收,库存无上限。
- 农场随累计学习无限扩大:小鸡数量 ≈ 累计学习天数,是"我走了多远"的具象化。
- **喂食互动**:可把蛋做成煎蛋/荷包蛋喂小鸡,小鸡开心转圈(纯情感互动,无数值,不喂**没有任何惩罚**)。
- **母鸡永远不会死**:缺勤时它蹲窝睡觉,回来时扑过来欢迎。
- 里程碑进化(不预告,达成才出现):累计 100/500/1000/3000/6000/9000 词,农场解锁新场景(鸡舍→果园→池塘→…),母鸡获得装饰(草帽、围巾…)。

### 3.2 图鉴(收集)

- 每个学会的词 = 一张卡片(词 + 图 + 音),按主题成套收集,套集齐有徽章。
- 图鉴只展示"已点亮",不展示"还差多少"。

### 3.3 连续天数(宽容版)

- streak 展示但**每月自动发 3 张补签卡**,病假旅行自动补;清零在产品上不存在——最低会回落到"本月最长纪录"。

### 3.4 考妈妈模式

- 每周日生成一张 10 题小测卷(本周新词 + 易忘词),皮皮当老师考妈妈,App 提供"老师阅卷界面"(皮皮判对错)。

### 3.5 家长周报

- 每周日给爸爸的 3 分钟摘要:本周新词、最易忘的 5 个词、streak 状态。v1 在 App 内"家长页"展示即可,推送后置。

## 4. 词库与学习内容

### 4.1 分层路线(按频率爬梯,孩子无感知)

| 阶段 | 年份 | 内容 | 累计词汇 |
|---|---|---|---|
| L1 | 1–2 年 | 小学高频 + 生活词(图片为主,不要求拼写) | ~1,500 |
| L2 | 3–4 年 | 初中词汇 + 通用高频,加例句和拼写 | ~4,000 |
| L3 | 5–7 年 | 高中词 → 学术词表(AWL)→ 托福核心 | 9,000+ |

### 4.2 词包(主题化投放)

- 首发:**「好吃的!」**(水果/蔬菜/饮料/食物,~120 词)——皮皮钦定。
- 已规划:「刀与妖怪」(鬼灭之刃主题:sword/demon/breathe/slash…)、「动物」「太空」「魔法城堡」。
- 选词原则:优先选**会在她生活中出现的词**(动画、超市、餐厅、游戏)。

### 4.3 词表来源(2026-07 定稿:权威开源词表为骨架)

- **L1 骨架**:义务教育英语课程标准(2022)小学阶段词汇(约 500 词)+ Fry 高频词表补生活词;
- **L2**:中考词表(约 1600);**L3**:高考词表(约 3500)→ 学术词表(AWL)→ 托福核心;
- 词表本身是公开事实清单;每个词的儿童释义、例句、主题包归类由 Claude 生成、爸爸抽查;
- 现有 87 词自动映射进新词库,学习记录不受影响;
- 投放顺序仍按主题包(§4.2),词表只决定"哪些词、什么层级",不决定呈现顺序。

### 4.4 单词卡片字段

word, 音标, 释义(中文,儿童语言), 本地图片资产 ID, 例句(简单), 例句翻译, 主题包, 频率层级(L1/L2/L3), 发音(v1 用 Web Speech API TTS,后期可换真人音频)。

## 5. 学习引擎(SRS + 题型系统)

### 5.1 调度

- 算法:**FSRS**(用 `ts-fsrs` 库),目标保留率 0.9(复习正确率 85–90%,"有点挑战但总能赢")。
- 每个词一张 FSRS 卡(不按题型分卡);答对 → Good,答错 → Again,不暴露 Easy/Hard。
- **复习债上限**:每日复习队列 ≤6(2026-07-17 爸爸定),按"最过期优先"取,其余自动顺延;**任何界面不显示积压总数**。
- 新词每日固定 4 个,复习负担过重(连续 3 天到期 >12,即积压超过两倍日容量)时自动暂停新词 1–2 天(对皮皮的文案:"今天母鸡想多陪陪老朋友")。

### 5.2 题型矩阵(2026-07 定稿:客观判分优先 + 手写贯穿)

设计动机:两键自评对 9 岁孩子数据易失真;选择题/听音题客观判分,手写默写用
"写完对照标准答案再自评"的**半客观**方式(有字迹可对照,作弊空间小),不用 OCR
(儿童手写识别不可靠,判错冤枉孩子的伤害大于收益)。

| 场景 | 题型 | 判分 |
|---|---|---|
| 新词·描红 | 四线三格淡灰字模,描一遍 | 不判分,过程即价值 |
| 新词·自测 / 复习·低熟练 | 选择题:看词 → 4 个中文选项选 1；确需图像辅助时使用 F4 本地单词插图 | 客观 |
| 复习·低熟练(与上交替) | 听音辨义:播放发音 → 选意思/图 | 客观 |
| 复习·较熟(stability 达阈值) | 默写:看中文 → 凭记忆手写英文 → 翻开对照 → 自评"写对了/没写对" | 半客观 |

- 选择题干扰项从同词包/同层级词里抽,避免太容易。
- 每词第一天写两遍(描红 + 收尾默写,见 §2.2);熟词默写复习每天上限 3 张
  (随复习队列 25→6 等比下调,防止 6 张复习里大半是手写导致超时)。
- 题型矩阵按年龄演进:5–6 年级加"中→英"产出方向;初中后加拼写键入、听写、例句填空。
- 手写技术:Canvas + Pointer Events(pen/touch 均可),`touch-action: none` 防误触;
  字迹存缩略图(收尾默写的字迹绑定当天的小鸡;图鉴展示"皮皮写的字")。

### 5.3 复习队列的安排与呈现

**安排:**
- 取词:到期卡"最过期优先"取 ≤6,其余顺延;
- 呈现顺序:**前 3 张选记忆强度最高的做热身**(开局连对入状态),
  其余乱序打散(拆开同日学的词,防"背顺序");
- 题型分配:stability < 7 天 → 选择题/听音辨义交替(同词不连续同题型,
  卡上记录 lastQuizType);stability ≥ 7 天 → 默写;
  **默写类复习每日上限 3 张**(随复习上限 6 等比下调),超出自动降级为选择题(控时阀门);
- 答错:FSRS 记 Again + 排队尾当场重考。

**呈现:**
- 复习与新词同一条卡片流、共用母鸡进度条,孩子无感知模块切换;
- 复习卡人设"老朋友",标签"还记得我吗?";
- **答错零惩罚感**:不出红叉不闪红,温和展示正确答案,配文案
  "这次见面来得正是时候!"(§2.7 记忆模型的产品化);
- 所有答题卡带"忘记了"按钮(§2.5):跳过进救援,不队尾重考;
- 积压总数任何界面不显示;首页只说"今天复习 X 个老朋友"(X≤6)。

## 6. 数据模型(IndexedDB via Dexie)

```
words        { id, word, ipa, meaning, imageAssetId?, imageUrl?, sentence, sentenceCn, pack, level }
cards        { wordId, state, due, stability, difficulty, reps, lapses, last_review }   // FSRS 状态
sessions     { date, newWords[], reviewed[], correctRate, durationSec, completed }
farm         { eggs, chicks, decorations[], scenesUnlocked[], henName }
meta         { streak, freezeCards, totalLearned, installDate, settings }
quizzes      { weekOf, wordIds[], momScore?, gradedByPipi }
seen         { wordId, lastSeenAt }   // 全系统共享的"上次见面时间",供陈旧度抽词(§2.4)
rescue       { wordId, capturedAt }   // 救援队列:被黄鼠狼"叼走"的词(§2.5)
```

- **数据安全(硬需求,2026-07-17 爸爸定稿方案)**:
  - 手动兜底:家长页一键导出/导入 JSON 备份(保留);
  - **云备份:自动推送到 GitHub 仓库**(与账本软件同方案)——每次打开 App,
    距上次备份 >24h 且有网 → 把备份 JSON 经 GitHub API 推到本仓库的
    `backups` 分支(不触发 Pages 构建);家长页提供"从云端恢复最新备份";
  - 访问令牌(fine-grained PAT,仅本仓库 contents 写权限)在家长页输入一次,
    **只存设备本地,绝不进代码**(仓库是公开的);
  - 仓库公开(爸爸定:本项目无隐私敏感数据)。Safari 可能清除站点数据,
    7 年记录不能只存本地,此方案是长期保障。

## 7. 技术架构

- **栈**:Vite + React + TypeScript + `vite-plugin-pwa`(Service Worker 离线)+ Dexie(IndexedDB)+ `ts-fsrs`。

## 8.5 F4 视觉交接清单（Claude 架构 / Codex 视觉）

F4 是当前唯一视觉母版，不再进行开放式“风格探索”。开始任务前必须按 [`../00-start/AI_START_HERE.md`](../00-start/AI_START_HERE.md) 的顺序读取产品、视觉、母版和协作文档。Claude 负责内部架构、状态和接口；Codex 负责真实资产、布局、动效、iPad 保真和视觉回归。任何 AI 都不得用自己的默认组件库或“相似日系素材”替换 F4。

风格基准：日系 Kawaii 手绘贴纸农场；全幅水粉背景、粗深棕手绘描边的独立透明角色/互动道具、奶油纸牌式学习 UI。贴纸是画风与层级，不是白色外框。Token 以 [`../02-visual/F4_VISUAL_SYSTEM.md`](../02-visual/F4_VISUAL_SYSTEM.md) 和锁定 F4 CSS 为准，不再以 `src/styles.css` 中的旧换肤值为设计来源。设备基准为 iPad Pro 11 英寸横屏 1194×834；其他横屏 iPad 只等比缩放核心舞台并延展背景。iPhone 与竖屏需要另行设计，不得自动重排 F4 后宣称完成适配。
