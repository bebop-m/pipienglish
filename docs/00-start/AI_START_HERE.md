## 固定共识与不可破坏项

- F4 已由小皮确认，不重新探索整体风格。
- “贴纸”指画风、独立透明素材和视觉层级，不指统一白边或贴纸收集册布局。
- 背景使用连续水粉空间；背景之外需要独立移动、换态、交互或摆放的角色、道具与装饰使用 F4 贴图语法，不能把背景的无描边水彩语言迁移到角色身上。
- 异色小鸡可以在任务许可内改变颜色、姿势、羽毛轮廓和局部造型，不要求只是机械换色；但必须与小皮、母鸡和现有小鸡保持同一 F4 角色家族。特殊小鸡可以有更明显差异，仍不得脱离该家族。
- 生产页面不是整张截图：背景/场景、独立物件、可动角色和 HTML/CSS UI 必须分层。
- 1194×834 是不可重排的核心坐标系；其他 iPad Pro 横屏只整体等比缩放并延展外围背景。
- 母版文件不可覆盖；新增和修订使用新文件名、新版本和新哈希。
- Claude 负责架构、数据、状态和接口；Codex 负责视觉、资产、布局、动效、iPad 保真和视觉回归。
- 未经爸爸/小皮批准的资产不得进入儿童生产界面。
- 今后每一个新建场景，不论编号和批次，都必须执行视觉系统 §9.4 的“建单与参考职责 → 候选定稿冻结 → 非破坏性技术整理 → F4 场景一可见轮廓比例 → 统一画布/锚点 → 仓库外独立校准 → 整体机械合成 → 批量授权/逐文件登记”串行门禁；不得把场景编号范围、赶进度或已有整图候选当成跳门理由。
- 每个场景都必须有一套免费小皮默认装束，与收费主题角色装扮分开；购买所有权全局有效，但实际穿戴按场景独立保存，切换场景不携带上一场景的选择。
- 场景 1 的 F4 独立鸡蛋模型只保留为旧场景兼容资产，不是今后新场景的生产母版。从场景 2 起，每个新场景都必须生成并冻结一组场景专属的完整鸡窝状态贴图，至少覆盖空窝、完整蛋和按孵化剩余时间分段的开裂/破壳状态；运行时整张切换，不得把 F4 独立鸡蛋机械叠到空窝上。
- 不得擅自实施“当前章＋下一章”四处活文档产品规格变更；历史第五处已归档且不得回改。该变更须爸爸另签并建立独立 F4-CHG。
- 用户脏改动和现存 worktree 不得覆盖、清理、移动、删除或 prune。

## 开始任务前四问

1. 本任务修改哪一层（背景、场景物件、角色、UI、架构/数据）？
2. 哪些文件/资产是直接参考，哪些不可修改，许可路径是什么？
3. 输出使用什么逻辑尺寸、画布、锚点或数据契约？
4. 用什么测试、截图、哈希或 diff 证明没有破坏 F4？

答不清四项时，不开始生成资产或修改代码。

## 面向视觉生成 AI 的资产交付

本轮不是自由创作。先确认当前任务只要求一张校准资产，实际打开视觉系统 §9.1 要求的最小锚图，写清身份/画风/环境/构图参考、允许变化、必须保留、透明画布、锚点和禁用元素。提交给生成工具前先复述每张参考图的唯一职责；场景图不得替代角色身份锚图。候选获批后按 §9.4 冻结具体源文件与哈希；透明化只改 alpha，归一只做等比缩放与锚点平移；先在仓库外审独立校准，再审冻结图层的机械合成。只有四个批准点全部通过后才可批量生产、逐文件验收和登记；候选与校准预览不得接入生产代码。
