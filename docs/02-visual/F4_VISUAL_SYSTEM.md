# 皮皮 English · F4 视觉系统（锁定版）

> 状态：**小皮已确认，作为后续全部视觉工作的母版。**  
> 本文描述的是 `design-samples/sticker-f-farm-v4.html` 已经呈现出来的风格，不是重新提出一种相近风格。

## 1. 裁决顺序

出现冲突时，按以下优先级裁决；越靠前权重越高：

1. 小皮对 F4 的确认和后续明确反馈；
2. `F4_BASELINE_MANIFEST.md` 登记的 F4 页面与原始图片；
3. 本文的视觉语法和验收规则；
4. `DESIGN_BRIEF.md`；
5. `SPEC.md` 中的产品与技术说明；
6. AI 自己对“日系”“Kawaii”“贴纸风”或“儿童 App”的理解。

不得用一张“看起来更精致”的新图覆盖风格母版。若新要求与母版冲突，先提出差异并等待确认。

## 2. 一句话定义

**日系 Kawaii 手绘贴纸农场：一张柔软、明亮的全幅水粉农场背景，承载带粗深棕手绘轮廓的独立透明角色与互动道具，再叠加奶油纸牌式、圆润且可落地的学习 UI。**

这里的“贴纸”是**画风和视觉层级**：角色轮廓清晰、像可以拿起来移动；不是给所有内容加白色外框，也不是把界面做成贴纸收集册。

两张原始参考母图位于 `docs/reference-images/`：角色参考板负责防止小鸡造型漂移；农场小鸡图负责最初的农场气质，并已由小皮确定为 PWA 桌面图标源母图。具体身份和哈希见 [`../reference-images/README.md`](../reference-images/README.md)。实际制作新资产时，当前 F4 对应角色 PNG 仍是直接身份母图。

## 3. 四层画面模型

所有新页面和新资产先确定所属层级，不允许把多个会独立变化的层画死在一张图里。

| 层级 | 内容 | 实现方式 | 关键规则 |
|---|---|---|---|
| L1 环境背景 | 天空、云、草地、谷仓、小路、远景栅栏和固定花草 | 全幅 RGB PNG/WebP | 无深色轮廓；手绘水粉质感；不承担点击和状态变化 |
| L2 场景物件 | 孵化棚、鸡舍、篮子、木牌、锅台、可更换装饰 | 独立透明 PNG/WebP | 与背景材质一致；需要点击、遮挡或换状态时必须独立 |
| L3 生命角色 | 小皮、母鸡、小鸡及动作状态 | 独立透明 PNG/WebP | 粗深棕手绘描边；统一五官、上色和画布锚点；可移动、镜像或换帧 |
| L4 交互 UI | 顶栏、任务纸牌、进度、按钮、气泡、弹层 | HTML/CSS + 少量本地位图 | 信息清楚、触控可达；奶油纸片感；不伪装成整张插画截图 |

白色或奶油色区域只用于确实需要承载文字和操作的 L4 控件。小鸡、母鸡、任务入口后面不得为了“贴纸风”统一垫一块白色贴纸底。

## 4. 形状、线条与材质语法

### 4.1 角色与可爱道具

- 外形以大圆形、软曲线、短肢体为主，允许轻微不对称和手绘抖动。
- 主轮廓为深棕黑，不用纯黑机械描边。F4 UI 基准色为 `#302925`；角色原图以现有图片里的实际轮廓为准。
- 轮廓粗、圆、略有粗细变化，不使用矢量软件默认的均匀描边。
- 眼睛是极简黑色椭圆或圆点；嘴、喙、腮红只保留识别所需的最少信息。
- 内部以暖色平涂和极淡的水粉明暗为主。禁止高光玻璃球、赛璐璐阴影、3D 体积光和塑料光泽。
- 角色内部可见轻微纸纹、色差和柔和晕染，但不能用统一噪点覆盖来冒充手绘。
- 角色资产不自带星星、花朵、蝴蝶结、文字气泡或落地光效；这些属于独立的短暂动效层。

### 4.2 背景与大型场景

- 天空、云、草地、谷仓和固定栅栏不使用角色同款深色粗描边。
- 背景是连续的水粉空间，不是几块几何色带，也不是由标准矢量图标拼成。
- 色彩明亮、温暖、柔和；允许局部有细腻色差，禁止大面积数字渐变和荧光色。
- 细节密度从近景向远景递减；固定花草用于丰富空间，但不能干扰角色拖动和点击。
- 新背景必须以 `farm-background-f3.png` 为直接风格参考；“相似农场照片”或通用卡通农场不构成参考。

### 4.3 UI

- UI 是清晰可实现的 HTML/CSS 组件，不把文字和按钮烘焙进背景图。
- 主要容器使用奶油纸、淡黄纸或白色纸牌；轮廓为 2–3px 深棕色，圆角大且略显柔软。
- 阴影短、暖、轻：表现纸牌被抬起，不表现悬浮玻璃或厚重塑料。
- 主要按钮使用温暖黄色，重要但不刺眼；次要状态使用粉、蓝、薄荷绿。
- 顶栏采用一组紧凑胶囊，展示今日单词、鸡蛋、小鸡、孵化状态和家长入口。
- 不使用标准后台面板、毛玻璃、霓虹、锐角卡片、全屏大白弹窗或满屏统一粗边框。

## 6. 角色比例与屏幕占位

以 1194×834 的 F4 画布为准，当前 HTML 角色容器是：

| 角色 | 基准容器 | 层级关系 |
|---|---:|---|
| 小皮 | 252×274 | 最大，农场主 |
| 母鸡 | 220×220 | 比小皮小、比小鸡明显大 |
| 小鸡 | 116×116 | 最小，可大量出现 |

比较的是资产在统一透明画布中的**可见主体和脚底基线**，不只比较原图像素。现有三张角色图均为 1254×1254 RGBA 方形画布；新动作不得通过改变透明留白偷偷改变角色的视觉大小。

角色身份必须保持：

- 小鸡：黄色圆身体、极简黑眼、橙色小喙、粉色腮红、头顶呆毛；不得突然增加鸡冠或复杂服装。
- 母鸡：与小鸡同一画法，红色三瓣鸡冠，体型更成熟但仍圆润幼拙。
- 小皮：草帽、蓝色背带裤、深色长发；细节不能比动物精致一个画风等级。

## 7. F4 首页构图锚点

以下是视觉母版的基准，不作为所有页面的通用布局：

- 核心画布：1194×834。
- 顶栏：上 18、左右 22，组件间距 7。
- 今日任务板：左 28、上 92、宽 390。
- 孵化棚：左 12、下 4、304×286。
- 救援篮入口：右 19、上 252、142×154。
- 小皮、母鸡和小鸡位于农场舞台层，可按规则自由移动；它们不是背景构图的一部分。
- 任务板、孵化棚、救援入口和顶栏是避障对象；透明留白不得形成看不见的“空气墙”。

首页禁止增加中间翻页线、角色背后的白色贴纸底、常驻“角色可自由散步”说明框，以及画死在农场里的厨房。

## 8. 动效与可动角色语法

- 散步是慢速、松弛的生活状态，不是游戏角色巡逻。单段位移为 9.2–15.8 秒，停留 2.6–5.6 秒。
- 行走起伏周期为 920ms，幅度小；不得高频弹跳。
- 小鸡默认分区活动并保持间距，避免总是聚成一团；数量增多时可逐步降低最小间距。
- 小鸡允许手动拖拽。拖动坐标、避障和回到局部散步区都必须在基准画布坐标系内计算。
- 母鸡、小鸡、小皮可偶尔挥手、轻跳、扭动、交谈；小皮可偶尔去孵化棚查看鸡蛋。
- 互动表情是短暂独立层，结束后删除；不得固化在角色 PNG 中。
- 必须支持 `prefers-reduced-motion` 和页面内动效开关。关闭后不影响信息和操作。

## 9. 独立图片制作协议

### 9.1 每次必须提供给生成 AI 的材料

生成新资产时，不得只发送一句风格形容词。至少提供：

1. 本文；
2. `docs/reference-images/chick-character-style-reference.png`（最初角色造型和画风来源）；
3. `docs/reference-images/pwa-icon-farm-master.png`（最初农场气质；仅图标任务把它作为身份母图）；
4. `design-samples/assets/chick-f3.png`（当前 F4 角色绘画语法和小鸡身份基准）；
5. `design-samples/assets/farm-background-f3.png`（当前 F4 环境材质和色彩基准）；
6. 与目标身份最接近的现有资产，例如母鸡新动作必须再提供 `mother-f3.png`；
7. 目标资产的结构化需求单；
8. 如资产需要放进具体界面，再提供 F4 全屏截图，说明它只负责比例和使用场景，不负责角色身份。

图片角色必须明确，不能简单说“参考附件”：

- **身份参考**：角色是谁，哪些特征不能变；
- **画风参考**：线条、上色、材质怎么画；
- **环境参考**：色温、背景材质和细节密度；
- **构图参考**：资产在界面里占多大、朝向哪里。

### 9.2 新资产需求单

```yaml
asset_id: chick-eating-f4-v1
layer: L3
subject: F4 小鸡
state_or_action: 低头吃一小口煎蛋
identity_reference: design-samples/assets/chick-f3.png
style_reference:
  - design-samples/assets/chick-f3.png
  - design-samples/assets/farm-background-f3.png
context_reference: F4 首页截图
canvas_px: 1254x1254
display_box_pt: 116x116
anchor: foot-center
facing: right
allowed_changes: 头部角度、翅膀姿势、喙部轻微张开
must_preserve: 身体比例、呆毛、眼睛、喙、腮红、描边和黄色色域
must_not_include: 白色贴纸边、文字、星星、花朵、蝴蝶结、投影、背景、额外食物
export: transparent PNG, RGBA, sRGB
```

### 9.3 通用生成提示词模板

> 以“身份参考图”保持同一个角色，以“画风参考图”保持 F4 的线条、配色与水粉质感。只改变需求单中 `allowed_changes` 指定的内容。输出一个可用于 iPad PWA 的独立透明贴图，主体完整、脚底锚点稳定、透明留白与现有同角色资产一致。使用粗而略有抖动的深棕手绘轮廓、柔和不透明色块、极淡水粉色差和极简五官。不要白色贴纸外框，不要背景，不要文字，不要装饰粒子，不要矢量感、3D、塑料高光、赛璐璐阴影或素材库卡通感。

英文模型可追加：

> Preserve the exact character identity, proportions, visible scale, hand-painted dark-brown outline, minimal facial features, pastel gouache fill, paper-like color variation and transparent-canvas alignment of the F4 identity reference. Change only the explicitly allowed pose. Export a clean RGBA cutout with no white sticker border, baked shadow, text, background or decorative particles. Avoid vector geometry, 3D gloss, cel shading and generic clip-art styling.

提示词不能保证一致性；最终是否进入项目只由第 10 节验收决定。

## 10. 新资产验收清单

每项都必须通过：

- 缩到目标显示尺寸后，第一眼仍像现有 F4 家族，而不是另一套“相似可爱”素材。
- 与身份母图叠加比较，头身比、五官距离、轮廓粗细和可见尺寸没有漂移。
- 透明画布、视觉中心、脚底锚点与同角色其他状态一致，换帧不跳动。
- 没有白边、方形底色、残留抠图毛边、孤立半透明像素或错误内阴影。
- 资产没有烘焙文字、数量、星花蝴蝶结、气泡或短期状态。
- 放入 `farm-background-f3.png` 上以 1× 目标尺寸检查：不突兀、不发灰、不显得过度锐利。
- 以 2× DPR 检查边缘；再以 1× 实际 iPad 显示尺寸检查，不只放大欣赏原图。
- 至少生成 2–4 个候选，由小皮或当前设计负责人选定，未选中的图不得自动成为新基准。

## 11. 明确拒绝项

- 用字符图标、通用图标、模板矢量图或素材库插画替代已确认角色。
- 给所有角色、卡片和按钮统一加白色贴纸外框。
- 把新装饰、角色和交互物件直接画进背景，导致无法换状态或移动。
- 为追求“精致”增加写实羽毛、鼻口结构、复杂反光、锐利线条和高频纹理。
- 让 UI 退化成一张带文字的整图截图，或让农场退化成普通网页背后的壁纸。
- 未经确认覆盖母版文件；所有修订使用新文件名和新版本号。
