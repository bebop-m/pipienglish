# 皮皮のEnglish · F4 视觉系统（锁定版）

> 状态：**小皮已确认，作为后续全部视觉工作的母版。**  
> 本文描述的是 `design-samples/sticker-f-farm-v4.html` 已经呈现出来的风格，不是重新提出一种相近风格。

## 1. 裁决顺序

出现冲突时，按以下优先级裁决；越靠前权重越高：

1. 小皮对 F4 的确认和后续明确反馈；
2. `F4_BASELINE_MANIFEST.md` 登记的 F4 页面与原始图片；
3. 本文的视觉语法和验收规则；
4. `DESIGN_BRIEF.md`；
5. `SPEC.md` 中的产品与技术说明；
6. AI 自己对“日系”“Kawaii”“贴纸风”或“儿童 App”的理解。

不得用一张“看起来更精致”的新图覆盖风格母版。若新要求与母版冲突，先提出差异并等待确认。

## 2. 一句话定义

**日系 Kawaii 手绘贴纸农场：一张柔软、明亮的全幅水粉农场背景，承载带粗深棕手绘轮廓的独立透明角色与互动道具，再叠加奶油纸牌式、圆润且可落地的学习 UI。**

这里的“贴纸”是**画风和视觉层级**：角色轮廓清晰、像可以拿起来移动；不是给所有内容加白色外框，也不是把界面做成贴纸收集册。

两张原始参考母图位于 `docs/reference-images/`：角色参考板负责防止小鸡造型漂移；农场小鸡图负责最初的农场气质，并已由小皮确定为 PWA 桌面图标源母图。具体身份和哈希见 [`../reference-images/README.md`](../reference-images/README.md)。实际制作新资产时，当前 F4 对应角色 PNG 仍是直接身份母图。

## 3. 四层画面模型

所有新页面和新资产先确定所属层级，不允许把多个会独立变化的层画死在一张图里。

| 层级 | 内容 | 实现方式 | 关键规则 |
|---|---|---|---|
| L1 环境背景 | 天空、云、草地、谷仓、小路、远景栅栏和固定花草 | 全幅 RGB PNG/WebP | 无深色轮廓；手绘水粉质感；不承担点击和状态变化 |
| L2 场景物件 | 孵化棚、鸡舍、篮子、木牌、锅台、可更换装饰 | 独立透明 PNG/WebP | 与背景材质一致；需要点击、遮挡或换状态时必须独立 |
| L3 生命角色 | 小皮、母鸡、小鸡及动作状态 | 独立透明 PNG/WebP | 粗深棕手绘描边；统一五官、上色和画布锚点；可移动、镜像或换帧 |
| L4 交互 UI | 顶栏、任务纸牌、进度、按钮、气泡、弹层 | HTML/CSS + 少量本地位图 | 信息清楚、触控可达；奶油纸片感；不伪装成整张插画截图 |

白色或奶油色区域只用于确实需要承载文字和操作的 L4 控件。小鸡、母鸡、任务入口后面不得为了“贴纸风”统一垫一块白色贴纸底。

## 4. 形状、线条与材质语法

### 4.1 角色与可爱道具

- 凡是背景之外需要独立移动、换态、交互、装扮或摆放的角色、道具与装饰，都使用 F4 贴图语法：独立透明素材、深棕手绘轮廓、柔和平涂和轻微水粉色差。这里的“贴图”不等于统一白边。
- 场景里的独立树木、路牌、鸡窝、贴纸装饰等即使不移动，只要需要单独摆放或遮挡排序，也按贴图资产处理；不得因为它们属于场景就改用背景的无描边水彩语言。
- 外形以大圆形、软曲线、短肢体为主，允许轻微不对称和手绘抖动。
- 主轮廓为深棕黑，不用纯黑机械描边。F4 UI 基准色为 `#302925`；角色原图以现有图片里的实际轮廓为准。
- 轮廓粗、圆、略有粗细变化，不使用矢量软件默认的均匀描边。
- 眼睛是极简黑色椭圆或圆点；嘴、喙、腮红只保留识别所需的最少信息。
- 内部以暖色平涂和极淡的水粉明暗为主。禁止高光玻璃球、赛璐璐阴影、3D 体积光和塑料光泽。
- 角色内部可见轻微纸纹、色差和柔和晕染，但不能用统一噪点覆盖来冒充手绘。
- 角色资产不自带星星、花朵、蝴蝶结、文字气泡或落地光效；这些属于独立的短暂动效层。

### 4.2 背景与大型场景

- 本节只适用于连续背景及明确烘焙在背景中的固定远景元素，不适用于独立角色、交互物件或可摆放装饰。
- 天空、云、草地、谷仓和固定栅栏不使用角色同款深色粗描边。
- 背景是连续的水粉空间，不是几块几何色带，也不是由标准矢量图标拼成。
- 色彩明亮、温暖、柔和；允许局部有细腻色差，禁止大面积数字渐变和荧光色。
- 细节密度从近景向远景递减；固定花草用于丰富空间，但不能干扰角色拖动和点击。
- 新背景必须以 `farm-background-f3.png` 为直接风格参考；“相似农场照片”或通用卡通农场不构成参考。

### 4.3 UI

- UI 是清晰可实现的 HTML/CSS 组件，不把文字和按钮烘焙进背景图。
- 主要容器使用奶油纸、淡黄纸或白色纸牌；轮廓为 2–3px 深棕色，圆角大且略显柔软。
- 阴影短、暖、轻：表现纸牌被抬起，不表现悬浮玻璃或厚重塑料。
- 主要按钮使用温暖黄色，重要但不刺眼；次要状态使用粉、蓝、薄荷绿。
- 顶栏采用一组紧凑胶囊，展示今日单词、鸡蛋、小鸡、孵化状态和家长入口。
- 不使用标准后台面板、毛玻璃、霓虹、锐角卡片、全屏大白弹窗或满屏统一粗边框。

## 6. 角色比例与屏幕占位

以 1194×834 的 F4 画布为准，当前 HTML 角色容器是：

| 角色 | 基准容器 | 层级关系 |
|---|---:|---|
| 小皮 | 252×274 | 最大，农场主 |
| 母鸡 | 220×220 | 比小皮小、比小鸡明显大 |
| 小鸡 | 116×116 | 最小，可大量出现 |

比较的是资产在统一透明画布中的**可见主体和脚底基线**，不只比较原图像素。现有三张角色图均为 1254×1254 RGBA 方形画布；新动作不得通过改变透明留白偷偷改变角色的视觉大小。

基础角色身份必须保持：

- 普通小鸡基准：黄色圆身体、极简黑眼、橙色小喙、粉色腮红、头顶呆毛。
- 母鸡：与小鸡同一画法，红色三瓣鸡冠，体型更成熟但仍圆润幼拙。
- 小皮：草帽、蓝色背带裤、深色长发；细节不能比动物精致一个画风等级。

章节小鸡变体遵守“同一家族，不必同一造型”：

- **异色小鸡**可以在结构化需求单许可范围内改变羽毛配色、姿势、朝向、头顶绒毛、翅尾轮廓、花纹和小幅身体比例；不要求只是对普通小鸡机械换色。
- **特殊小鸡**可以有更明显的轮廓、主题花纹或经批准的小型特征，但仍必须保留 F4 家族的圆润比例、极简五官、深棕手绘轮廓、柔和平涂与幼拙气质。
- 任何变体都不能迁移成背景式无描边水彩、写实羽毛、3D、赛璐璐或另一套“相似可爱”角色。若需求单没有允许某项造型变化，就按最接近的身份母图保留。

以下是视觉母版的基准，不作为所有页面的通用布局：

- 核心画布：1194×834。
- 顶栏：上 18、左右 22，组件间距 7。

- 孵化棚：左 12、下 4、304×286。
- 救援篮入口：右 19、上 252、142×154。

首页禁止增加中间翻页线、角色背后的白色贴纸底、常驻“角色可自由散步”说明框，以及画死在农场里的厨房。

## 8. 动效与可动角色语法

- 散步是慢速、松弛的生活状态，不是游戏角色巡逻。单段位移为 9.2–15.8 秒，停留 2.6–5.6 秒。
- 行走起伏周期为 920ms，幅度小；不得高频弹跳。
- 小鸡默认分区活动并保持间距，避免总是聚成一团；数量增多时可逐步降低最小间距。
- 小鸡允许手动拖拽。拖动坐标、避障和回到局部散步区都必须在基准画布坐标系内计算。
- 母鸡、小鸡、小皮可偶尔挥手、轻跳、扭动、交谈；小皮可偶尔去孵化棚查看鸡蛋。
- 互动表情是短暂独立层，结束后删除；不得固化在角色 PNG 中。
- 必须支持 `prefers-reduced-motion` 和页面内动效开关。关闭后不影响信息和操作。

## 9. 独立图片制作协议

### 9.1 每次必须提供给生成 AI 的材料

生成新资产时，不得只发送一句风格形容词，也不得为了省上下文跳过身份锚图。所有任务都必须提供本文 §9.1–§10、目标资产的结构化需求单，并按下表**实际打开**最小必要锚图：

| 目标资产 | 必须打开的最小锚图 | 参考职责 |
|---|---|---|
| 小鸡动作、异色或特殊小鸡 | `design-samples/assets/chick-f3.png`；`docs/reference-images/chick-character-style-reference.png` | 前者是当前 F4 身份家族与绘画语法基准；后者只防止造型和画风漂移 |
| 母鸡动作或装扮 | `design-samples/assets/mother-f3.png`；需要同族校验时再开 `chick-f3.png` | 母鸡身份、比例和线条基准；小鸡图只校验角色家族关系 |
| 小皮动作或装扮 | `design-samples/assets/xiaopi-f3.png` | 小皮身份、比例、线条和细节等级基准 |
| 新背景 | `design-samples/assets/farm-background-f3.png`；`docs/reference-images/pwa-icon-farm-master.png` | 前者是环境材质与色温直接基准；后者只负责最初农场构图气质，不负责角色画风 |
| 独立场景道具或装饰 | 目标场景的已批准背景；最接近的现有 F4 贴图资产 | 背景只给环境关系；现有贴图负责轮廓、材质和独立资产语法 |

如资产要放进具体界面，再提供对应 F4 全屏截图；截图只负责比例、朝向、遮挡和使用场景，默认不负责角色身份或角色画风。没有目标身份锚图时不得用背景、场景总览或通用参考图代替。

图片角色必须明确，不能简单说“参考附件”：

- **身份参考**：角色是谁，哪些特征不能变；
- **画风参考**：线条、上色、材质怎么画；
- **环境参考**：色温、背景材质和细节密度；
- **构图参考**：资产在界面里占多大、朝向哪里。

### 9.2 新资产需求单

```yaml
asset_id: chick-eating-f4-v1
layer: L3
subject: F4 小鸡
state_or_action: 低头吃一小口煎蛋
identity_reference: design-samples/assets/chick-f3.png
style_reference:
  - design-samples/assets/chick-f3.png
environment_reference: design-samples/assets/farm-background-f3.png
composition_reference: F4 首页截图（只负责显示比例、朝向与使用位置）
canvas_px: 1254x1254
display_box_pt: 116x116
anchor: foot-center
facing: right
allowed_changes: 头部角度、翅膀姿势、喙部轻微张开
must_preserve: 身体比例、呆毛、眼睛、喙、腮红、描边和黄色色域
must_not_include: 白色贴纸边、文字、星星、花朵、蝴蝶结、投影、背景、额外食物
export: transparent PNG, RGBA, sRGB
```

若目标是章节异色或特殊小鸡，`identity_reference` 表示必须属于同一 F4 角色家族，不代表轮廓必须逐像素等同普通小鸡。允许变化必须逐项写进 `allowed_changes`；未列出的身份特征继续由 `must_preserve` 锁定。场景总览图不得同时填写为角色的 `identity_reference` 或 `style_reference`。

### 9.3 通用生成提示词模板

> 以“身份参考图”保持同一个角色，以“画风参考图”保持 F4 的线条、配色与水粉质感。只改变需求单中 `allowed_changes` 指定的内容。输出一个可用于 iPad PWA 的独立透明贴图，主体完整、脚底锚点稳定、透明留白与现有同角色资产一致。使用粗而略有抖动的深棕手绘轮廓、柔和不透明色块、极淡水粉色差和极简五官。不要白色贴纸外框，不要背景，不要文字，不要装饰粒子，不要矢量感、3D、塑料高光、赛璐璐阴影或素材库卡通感。

英文模型可追加：

> Preserve the exact character identity, proportions, visible scale, hand-painted dark-brown outline, minimal facial features, pastel gouache fill, paper-like color variation and transparent-canvas alignment of the F4 identity reference. Change only the explicitly allowed pose. Export a clean RGBA cutout with no white sticker border, baked shadow, text, background or decorative particles. Avoid vector geometry, 3D gloss, cel shading and generic clip-art styling.

提示词不能保证一致性；最终是否进入项目只由第 10 节验收决定。

### 9.4 所有新建场景的永久生产门禁

本节适用于本软件今后创建的**每一个新场景**，不受场景编号、章节批次或当前路线图限制。场景二只是在 2026-07-22 验证出本流程的实例，不构成适用范围上限。除非爸爸明确批准一份写明差异、影响和回退方式的变更记录，任何新场景都必须严格按以下顺序串行推进，不得合并、倒置或跳过门禁。新场景的免费默认装束、场景独立穿戴和场景专属孵化鸡窝状态接口同时遵守 [`F7_SCENE_STICKERS_WARDROBE_GATE_AND_TEST_PLAN.md`](../05-architecture/F7_SCENE_STICKERS_WARDROBE_GATE_AND_TEST_PLAN.md) 的最新确认定稿。

#### A. 建单与参考职责冻结

1. 先建立场景资产清单，逐项标明：背景、可交互设施、普通角色、异色角色、特殊角色、场景专属孵化鸡窝状态组、贴纸、免费小皮场景默认装束、独立的收费主题角色装扮，以及必要的 `back/body/front` 或状态帧。原创项数量与实际交付文件数分别记录，不能用“共 N 张图”掩盖分层或状态文件，也不得把免费默认装束与收费商品合并计数。
2. 每项先填写 §9.2 需求单并实际打开 §9.1 的最小锚图。每张参考图只能承担已声明的职责：身份、画风、环境或构图。场景总览、孵化矩阵、故事板、动画状态表和整屏截图默认只有构图、比例、锚点或状态顺序价值；未经明确指定，**没有角色身份或角色画风参考价值**。
3. 先查现有冻结资产。普通角色和明确批准共享的非孵化资产能够复用时，必须直接复用；不得为了“更贴合新场景”重新生成一个近似版本。但场景 1 的 F4 独立鸡蛋、裂纹和蛋壳不再属于新场景共享源；新场景必须按自己的鸡窝造型生成并冻结完整状态组。若爸爸批准的是状态矩阵，则矩阵的每一格就是状态资产本身，只能整格派生，不得拆出鸡蛋后替换、调形或重新组合。
4. 孵化鸡窝需求单必须列出空窝、完整蛋、按剩余时间分段的一个或多个开裂状态、破壳状态和需要的出壳结果。每一状态都是该场景鸡窝的完整贴图，必须分别经过候选批准、冻结和逐文件验收；运行时只按状态整张切换，不得在空窝上临时叠放 F4 独立鸡蛋或裂纹/蛋壳图层。逐格技术整理只允许白底透明化、整格等比缩放、统一画布和 anchor，不得局部调整格内任何主体。

#### B. 候选生成与定稿冻结

1. 生成阶段一次只解决一项身份或一组明确关联的状态。候选必须使用新文件名并留在候选区或仓库外；不得覆盖母版、不得接入生产目录、不得被代码或通配符发现。
2. 爸爸/小皮必须按具体候选明确批准。批准后立即冻结：原始文件路径、用途、批准名称/字母、像素尺寸、色彩模式、原始 SHA-256、直接参考图、允许变化与必须保留项。口头描述不能替代具体文件；“同类风格”“参考定稿重做”不等于复用定稿。
3. 从冻结时起，该文件是后续校准和批量生产的唯一像素来源。不得重新生成、重画、补画、概括、风格迁移或用生成式整图合成把它画一遍。若定稿确有缺陷，必须停止并重新回到候选批准门，而不是在校准阶段偷偷修图。

#### C. 非破坏性技术整理

1. 定稿缺少透明背景时，先从冻结源文件派生独立透明版本；只允许改变 alpha/透明边缘，不改变主体内部 RGB、造型、颜色、纹理或线条。原文件必须保留，派生文件登记来源哈希与自身哈希。
2. 技术验收至少证明：主体内部 RGB 与冻结源一致、四角透明、无方形底色、无背景残留、无孤立半透明像素、无轮廓缺损。若无法在不改主体像素的前提下干净分离，停止并请求重新批准合适源文件。
3. 透明化、裁边分析、统一画布、缩放、锚点移动和分层合成都必须是可复现的确定性处理；不得用图像生成模型承担这些步骤。

#### D. 对照 F4 场景一的比例基线

所有新场景先在 1194×834 逻辑舞台对照场景一，再谈新场景构图。**比较可见 alpha 轮廓，不把 HTML 容器尺寸、1254 方形源画布或透明留白误当成角色大小。**当前冻结基线为：

| 角色 | 场景一容器 | 1254×1254 源图 alpha bbox | 源图可见轮廓 | 映射后可见轮廓（约） |
|---|---:|---:|---:|---:|
| 小皮 `xiaopi-f3` | 252×274 | `(343,116)–(973,1104)` | 630×988 | 127×199 |
| 母鸡 `mother-f3` | 220×220 | `(258,153)–(987,1058)` | 729×905 | 128×159 |
| 普通小鸡 `chick-f3` | 116×116 | `(336,211)–(905,983)` | 569×772 | 53×71 |

新场景角色、鸡窝、驿站、孵化设施和其他交互设施都必须以这些可见轮廓及场景一设施占位为尺度参照；设施不得因为源图留白少或候选图画得满而变成前景主视觉。确需改变角色或设施的叙事尺度时，必须先提交带可见 bbox 数值的例外对比图并获得爸爸明确批准。

#### E. 统一画布、等比缩放与锚点归一

1. 同一角色家族或状态组使用其冻结身份母图的画布、display box、视觉中心和 foot/ground anchor。小鸡普通、异色和特殊款统一以 `chick-f3` 的 1254×1254 画布、普通小鸡脚底基线和 116×116 scene-one display box 为归一目标。
2. 只允许等比缩放、平移和经批准的整组镜像；禁止非等比拉伸、局部变形、裁掉主体或用透明留白伪造大小。宽体/高体特殊造型使用 contain 进入目标可见框，并保持自身比例；由此产生的自然高矮差异必须在校准板上显式标出。
3. 所有分层文件使用完全一致的 canvas、缩放、视觉中心、foot/ground anchor 与 display box；移动、缩放和镜像只施加到整组。同一场景的鸡窝状态贴图必须使用完全一致的 canvas、鸡窝轮廓、display box 和放置位置，只允许孵化状态所需的鸡蛋、裂纹、蛋壳或出壳角色变化；状态切换不得导致鸡窝跳位、缩放或轮廓漂移。

#### F. 仓库外两级校准审图

1. **独立校准门：**先在仓库外输出每个冻结素材的独立透明版、统一画布版和并排校准板。校准板必须标出可见 bbox、统一脚底/地面基线、缩放系数和目标 display box，让爸爸确认原造型未变且相对 F4 比例正确。
2. 独立校准未批准，不得制作整体审图。批准后才可在已批准背景上做**机械分层合成**：精确摆放冻结像素图层，检查角色、设施、遮挡、层级、巢心/地面锚点与视觉重心。
3. 整体审图禁止使用生成式整图合成，禁止把已批准贴图重新画进场景，禁止用此前失败的生成图补构图。预览、脚本、中间透明图和校准报告继续留在仓库外，直至爸爸明确批准写入生产资产。

#### G. 批量生产、逐文件验收与登记

1. 只有在“候选定稿冻结”“独立比例/锚点校准”“整体机械合成”三门均得到爸爸/小皮明确批准后，才可请求并获得批量生产授权；批准一张校准图不自动等于批准批量、接入或发布。
2. 批量生产只能使用已冻结清单、冻结源文件和已批准的确定性参数。任何新增造型、新状态、新分层或参数漂移都退回对应门禁，不得顺手扩产。
3. 每个交付文件逐项执行第 10 节验收，并登记：稳定逻辑 ID、文件名、用途、来源文件及 SHA-256、派生方式、像素尺寸、色彩模式、alpha bbox、display box、foot/ground/center anchor、层级顺序、批准状态与最终 SHA-256。每个场景还必须登记免费小皮默认装束映射，以及该场景鸡窝每张完整状态贴图对应的业务状态、剩余时间分段和稳定资产 ID。
4. 未逐文件验收不得标记 `production`；未登记 `production` 不得复制到生产资产目录、写入清单、进入离线包或由代码/通配符加载。批量完成后还必须以 1194×834 生产舞台做一次实际尺寸机械合成回归。

#### H. 四个不可越过的批准点

| 门禁 | 爸爸/小皮看到的材料 | 批准后才允许 |
|---|---|---|
| G1 候选定稿 | 具体候选文件与身份/画风核对 | 冻结原始文件和哈希 |
| G2 独立校准 | 透明版、统一画布、三类并排或同组对比、bbox/anchor/scale | 制作整体机械合成预览 |
| G3 整体校准 | 冻结图层在目标背景中的精确合成 | 冻结批量参数与交付清单 |
| G4 批量授权 | 最终清单、参数、生产目标与回退点 | 批量生产、逐文件验收；接入仍须完成登记 |

任一批准点没有明确答复都视为未批准。时间紧、候选“看起来差不多”、已有整图效果图或后续场景数量很多，都不能作为跳门理由。

## 10. 新资产验收清单

每项都必须通过：

- 缩到目标显示尺寸后，第一眼仍像现有 F4 家族，而不是另一套“相似可爱”素材。
- 同一角色动作或装扮与身份母图叠加比较，确认未获准的头身比、五官距离、轮廓粗细和可见尺寸没有漂移；异色/特殊变体按需求单核对允许变化，并确认未变化部分仍属于同一 F4 家族。
- 透明画布、视觉中心、脚底锚点与同角色其他状态一致，换帧不跳动。
- 没有白边、方形底色、残留抠图毛边、孤立半透明像素或错误内阴影。
- 资产没有烘焙文字、数量、星花蝴蝶结、气泡或短期状态。
- 放入 `farm-background-f3.png` 上以 1× 目标尺寸检查：不突兀、不发灰、不显得过度锐利。
- 以 2× DPR 检查边缘；再以 1× 实际 iPad 显示尺寸检查，不只放大欣赏原图。
- 至少生成 2–4 个候选，由小皮或当前设计负责人选定，未选中的图不得自动成为新基准。
