# F4 美术锚点路由修复与小鸡变体口径

## 1. 触发原因

开发减负方案没有修改 F4 母版、生产图片、哈希或渲染代码，但新版 `AI_START_HERE.md` 将新美术任务路由到视觉系统 §9.2/§10，遗漏了 §9.1 的强制锚图清单，并把原先“实际打开哪些图片”的要求缩写为“直接参考图说明”。

该缺口允许执行者把章节场景总览同时当成角色画风参考，导致背景的无描边水彩语言迁移到小鸡身上。由此生成的暖杏焦糖、青绿蜂蜜、珊瑚淡紫三张仓库外预览全部作废：不进入候选、仓库、基准或验收记录。

## 2. 最终视觉裁决

- **背景层**使用连续水粉/水彩空间；固定远景不使用角色同款深色粗描边。
- **背景之外的独立元素**——小皮、母鸡、小鸡、鸡窝、道具和可摆放装饰——使用 F4 贴图语法：透明独立素材、深棕手绘轮廓、柔和平涂与轻微水粉色差。“贴图”不等于统一白边。
- **异色小鸡**不要求只是对普通小鸡机械换色。需求单可以允许颜色、姿势、朝向、羽毛轮廓、花纹和局部造型变化。
- **特殊小鸡**可以有更明显的轮廓、主题花纹或批准特征。
- 所有小鸡变体都必须与现有小鸡、母鸡和小皮保持同一 F4 角色家族；不得变成背景式水彩、写实羽毛、3D、赛璐璐或另一套“相似可爱”素材。

## 3. 路由修复

新美术资产任务现在必须：

1. 阅读 `F4_VISUAL_SYSTEM.md` §9.1–§10，不得从 §9.2 开始；
2. 按目标类型实际打开最小身份锚图；
3. 在 `CURRENT_TASK.md` 与结构化需求单中分别填写身份、画风、环境、构图参考和允许变化；
4. 在生成前复述每张参考图的唯一职责；
5. 将场景总览、界面截图和背景限制为环境/构图参考，不得用它们替代角色身份锚图；
6. 缺少身份锚图、参考职责冲突或允许变化不明确时立即停止。

该修复保留减负原则：代码/数据任务仍不读取 PNG；视觉任务只打开与目标直接相关的最小锚图，不恢复所有任务无差别加载整套图片。

## 4. 资产与产品影响

- 未修改 `design-samples/`、`public/`、`docs/reference-images/` 或任何 PNG。
- 未修改 `F4_BASELINE_MANIFEST.md`，所有锁定哈希保持不变。
- 未修改 `src/`、儿童界面、第二场景主题或章节产品规则。
- 本变更只修正文档路由与视觉契约；下一张校准图仍需爸爸/小皮单独批准。

## 5. 验证边界

- `git diff --name-only` 必须仅命中本记录与任务许可的四份现行文档。
- `git diff --quiet -- public design-samples src docs/reference-images docs/02-visual/F4_BASELINE_MANIFEST.md` 必须通过。
- 文档检索必须同时命中：§9.1–§10、美术四类参考职责、背景/贴图分界、异色可变造型和“场景图不得替代身份锚图”。
