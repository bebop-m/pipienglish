## 固定审批流程

1. Codex 用已批准的 F4 本地资产制作一个独立视觉候选。
2. 输出 1194×834 PNG，并列明所用真实状态与按钮。
3. 爸爸把图片交给小皮；小皮回复“可以”或指出修改点。
4. 只有标为 `xiaopi-approved` 后，Codex 才把该状态接入生产 React、真实 VM 和自动测试。
5. 完成一屏后再进入下一屏；不得用后续页面反向迫使小皮接受前一屏。

候选文件位于 `design-samples/stage-h/`，截图位于 `visual-regression/stage-h-candidates/`。候选只用于审批，不是生产页面。

已批准生产组件的回归截图位于 `visual-regression/stage-h-production/`。H-1 的生产组件已落地，但孩子可见导航继续门控，下一项审批为 H-2 描红卡。
